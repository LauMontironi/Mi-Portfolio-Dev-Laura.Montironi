🚀 Desarrollo de Aplicación Full-Stack – Gestor Familiar de Eventos

Este proyecto consiste en una aplicación web para la gestión de familias, miembros y eventos, desarrollada con arquitectura full-stack, autenticación segura y base de datos relacional en la nube.

🗄️ Backend y Base de Datos

  • Base de datos relacional en SQL con relaciones entre usuarios, familias, miembros y eventos.
  • Base de datos desplegada en servidor online Railway.
  • API REST desarrollada con FastAPI.
  • CRUD completo con peticiones GET, POST, PUT y DELETE.
  • Validación de datos usando Pydantic.
  • Conexión asíncrona con aiomysql.
  • Gestión de entorno virtual y dependencias.

🔐 Seguridad y Autenticación

  • Hash de contraseñas con passlib y algoritmo Argon2.
  • Autenticación con OAuth2 + JWT.
  • Dependencias de seguridad en FastAPI para proteger rutas privadas.
  • Cada usuario solo puede acceder a los datos de su familia.

🎨 Frontend con Angular

  • Aplicación desarrollada con Angular.
  • Navegación con Angular Router.
  • Formularios con FormsModule.
  • Two-way data binding.
  • Comunicación entre componentes con @Input() y @Output().
  • Uso de Signals.
  • Validación de formularios en cliente.
  • Consumo de API mediante HTTP Client.
  • Control flow moderno con @if, @for, @switch.
  • Uso de variables como $index y $even.
  • Formateo de fechas con Date Pipe.

🧠 Aprendizajes Clave

  • La base de datos debía estar en un servidor online para que el frontend funcionara.
  • Tuve que reducir la sobre-normalización para simplificar el consumo desde el frontend.
  • Al registrarse, el usuario debe crear su familia base.
  • Los selectores deben filtrar por familia del usuario autenticado.
  • Si el usuario ya tiene familia, entra directamente sin volver a crearla.

Arquitectura Técnica

API REST diseñada por módulos, con seguridad por JWT y acceso restringido por familia (ownership). Estructura por capas (routes → controllers → services) y validación con Pydantic.

AUTH Login + JWT
POST /auth/login

Autenticación con JWT + hash Argon2. Protege rutas privadas con dependencias de seguridad.

FAMILIES Crear / leer familia
POST /families
GET /families/{family_id}

Creación del núcleo familiar inicial y carga de datos base.

MEMBERS Miembros por familia
GET /fmembers/families/{family_id}

Devuelve miembros filtrados por familia del usuario autenticado.

EVENTS Eventos (CRUD)
GET /events/families/{family_id}
POST /events/families/{family_id}
DELETE /events/families/{family_id}/events/{event_id}

Gestión completa de eventos. Permisos aplicados por ownership y rol.

DATA SQL + Relaciones

Modelo relacional: users, families, family_members, children, events. Consultas optimizadas para dashboard y filtros por familia.

DEPLOY Frontend + Backend

Frontend desplegado en Vercel. Backend y base de datos en Railway. Configuración de variables de entorno y CORS.
