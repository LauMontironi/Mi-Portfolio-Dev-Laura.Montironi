Full Stack Project

🎓 SaaS LMS Platform

Plataforma full-stack de cursos online con panel de alumno y panel admin, conectando Angular + FastAPI + PostgreSQL en una arquitectura real orientada a producto.

Frontend Code Backend Code API Docs

Qué es

SaaS LMS es una plataforma de aprendizaje online donde un estudiante puede registrarse, iniciar sesión, explorar cursos, inscribirse, seguir su progreso y marcar lecciones como completadas. Un administrador puede gestionar cursos y lessons desde un panel interno con estadísticas reales.

Funcionalidades clave

👩‍🎓 Panel de alumno
  • Registro y login con JWT + hashing de contraseñas
  • Dashboard con cursos recomendados, trending y catálogo completo
  • Continue learning con progreso real
  • Enroll en cursos con un click
  • Course detail con video embebido y listado de lessons
  • Marcar lecciones como completadas / pendientes
  • Barra de progreso por curso
  • Perfil con datos del usuario y cursos inscritos
  • Avatar por URL con fallback a iniciales
🛠️ Panel de admin
  • Acceso protegido por rol (solo admins)
  • Stats generales de la plataforma
  • CRUD completo de cursos
  • CRUD de lessons por curso
  • Badges de estado, nivel y featured en tabla
  • Redirección automática según role al hacer login

Stack & Arquitectura

🅰️ Frontend — Angular 21
  • Standalone components + signals + computed()
  • Lazy loading por ruta
  • Interceptor JWT automático
  • Guards por autenticación y por rol
  • PrimeNG UI + skeletons de carga
  • Animaciones CSS en cards y secciones
  • Desplegado en Netlify
🐍 Backend — FastAPI
  • FastAPI async + SQLAlchemy + PostgreSQL
  • Pydantic para validación y schemas
  • Arquitectura por capas: routes / controllers / schemas / models
  • Endpoints de enrollment y lesson progress
  • Auth con JWT + bcrypt
  • Desplegado en Railway

Decisiones técnicas interesantes

  • Separación clara entre panel de alumno y panel admin mediante guards por rol
  • Redirección del frontend según el role real obtenido desde /users/me al hacer login
  • Signals y computed() de Angular 21 en lugar de RxJS para estado local
  • Skeletons de carga en todas las páginas para mejorar UX percibida
  • Continue learning incluye cursos con 0% de progreso (recién inscritos)
  • Progreso calculado en backend en base a lecciones completadas / total
  • Interceptor JWT automático — no hay que añadir headers manualmente en cada llamada

Lo que aprendí construyéndolo

Este proyecto me ayudó a trabajar con una arquitectura full-stack mucho más real: no solo crear vistas, sino conectar auth, backend, base de datos, paneles por rol, lógica de negocio y gestión de contenido desde cero.

También fue una buena práctica para pasar de pantallas mockeadas a pantallas conectadas de verdad, mantener componentes reutilizables y trabajar con un flujo más cercano al de un producto real. Aprendí a gestionar estado con signals, a proteger rutas por rol tanto en frontend como en backend, y a estructurar una API con capas bien separadas.
